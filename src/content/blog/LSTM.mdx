## RNN, 무엇이 문제였나

지난 글에서 RNN의 핵심 한계를 확인했습니다. 역전파가 시간축을 따라 거슬러 올라가며 기울기가 곱해지는데, 그 인수가 1보다 작으면 기울기는 **지수적으로 소실**됩니다.

$$
\frac{\partial L}{\partial h_0} = \frac{\partial L}{\partial h_T} \cdot \prod_{t=1}^{T} W_{hh} \cdot \tanh'(z_t)
$$

결과적으로 시퀀스가 조금만 길어져도 초반 토큰의 가중치는 거의 업데이트되지 않아, **먼 과거의 정보를 기억하는 것이 사실상 불가능**해집니다.

이것이 RNN이 "장기 의존성(long-term dependency)"에 취약한 이유입니다.

## LSTM 등장 배경

1997년 Sepp Hochreiter와 Jürgen Schmidhuber는 이 문제를 구조적으로 해결하는 논문 **"Long Short-Term Memory"** 를 발표합니다.

핵심 아이디어는 이렇습니다.

> **은닉 상태 하나에 모든 것을 맡기지 말고, 장기 기억을 위한 별도 통로를 만들자.**

RNN이 은닉 상태 $h_t$ 하나로 모든 정보를 압축했다면, LSTM은 두 가지 상태를 분리해 운용합니다.

## 두 가지 상태: 셀 상태와 히든 상태

| 상태 | 기호 | 역할 | 비유 |
|------|------|------|------|
| **셀 상태** | $c_t$ | 장기 기억 | 컨베이어 벨트 |
| **히든 상태** | $h_t$ | 단기 기억 / 출력 | 작업 메모리 |

### 셀 상태 $c_t$ — 컨베이어 벨트

셀 상태는 시퀀스 전체를 가로질러 흐르는 **고속도로**입니다. 게이트가 허락하지 않는 한 값이 거의 그대로 전달되며, 이 덕분에 기울기가 멀리까지 흘러갈 수 있습니다.

컨베이어 벨트 위의 물건처럼 — 누군가 일부러 내려놓거나 새 물건을 올리지 않는 한 그냥 흘러갑니다.

### 히든 상태 $h_t$ — 작업 메모리

히든 상태는 매 타임스텝마다 셀 상태를 "필터링"해서 만들어지는 **출력 신호**입니다. 다음 시점의 계산에도 참여하지만, 셀 상태보다 훨씬 빠르게 바뀝니다.

## 세 개의 게이트

LSTM은 셀 상태로 흐르는 정보를 **게이트(gate)** 로 제어합니다. 게이트는 $\sigma$ (시그모이드) 함수를 사용해 0~1 사이 값을 출력하며, 이 값이 "밸브"처럼 정보의 흐름을 조절합니다.

$$
\sigma(z) = \frac{1}{1 + e^{-z}} \in (0, 1)
$$

### 잊기 게이트 (Forget Gate) $f_t$

$$
f_t = \sigma(W_f \cdot [h_{t-1},\, x_t] + b_f)
$$

이전 셀 상태 $c_{t-1}$을 **얼마나 기억할지** 결정합니다.

- $f_t \approx 1$ → 거의 전부 유지 (기억)
- $f_t \approx 0$ → 거의 전부 삭제 (망각)

예시: "The **cat**, which ate the fish, **was** hungry." 에서 동사 "was"를 처리할 때 주어 "cat"의 단수 정보를 유지해야 합니다. 이 정보는 잊기 게이트가 높게 유지시켜줍니다.

### 입력 게이트 (Input Gate) $i_t$ & 후보 셀 $g_t$

$$
i_t = \sigma(W_i \cdot [h_{t-1},\, x_t] + b_i)
$$

$$
g_t = \tanh(W_g \cdot [h_{t-1},\, x_t] + b_g)
$$

두 값이 함께 작동해 셀 상태에 **새로운 정보를 추가**합니다.

- $i_t$ : 새 정보를 얼마나 쓸지 (게이트 — 0~1)
- $g_t$ : 어떤 내용을 쓸지 (후보값 — −1~1)

### 셀 상태 업데이트

$$
c_t = f_t \odot c_{t-1} + i_t \odot g_t
$$

컨베이어 벨트에 빗대면:

- $f_t \odot c_{t-1}$ → 기존 물건을 $f_t$ 비율로 남기고
- $i_t \odot g_t$ → 새 물건을 $i_t$ 비율로 올린다

$\odot$ 는 원소별(element-wise) 곱입니다.

### 출력 게이트 (Output Gate) $o_t$ & 히든 상태 $h_t$

$$
o_t = \sigma(W_o \cdot [h_{t-1},\, x_t] + b_o)
$$

$$
h_t = o_t \odot \tanh(c_t)
$$

셀 상태 전체를 그대로 내보내는 것이 아니라, $o_t$가 결정한 만큼만 꺼내 $h_t$를 만듭니다. 셀 상태에서 현재 시점에 필요한 정보만 추출하는 역할입니다.

## 수식 한눈에 보기

| 수식 | 이름 | 활성화 | 역할 |
|------|------|--------|------|
| $f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)$ | 잊기 게이트 | $\sigma$ | 이전 기억 유지 비율 |
| $i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)$ | 입력 게이트 | $\sigma$ | 새 정보 수용 비율 |
| $g_t = \tanh(W_g [h_{t-1}, x_t] + b_g)$ | 후보 셀 | $\tanh$ | 새롭게 기록할 내용 |
| $c_t = f_t \odot c_{t-1} + i_t \odot g_t$ | 셀 상태 | — | 장기 기억 업데이트 |
| $o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)$ | 출력 게이트 | $\sigma$ | 출력할 정보의 양 |
| $h_t = o_t \odot \tanh(c_t)$ | 히든 상태 | — | 단기 기억 / 출력 |

## 왜 기울기 소실이 줄어드나?

RNN에서는 기울기가 $W_{hh}$와 $\tanh'$의 곱을 반복해서 통과해야 했습니다.

LSTM에서는 셀 상태 $c_t$로 흐르는 경로가 있습니다.

$$
\frac{\partial c_t}{\partial c_{t-1}} = f_t
$$

잊기 게이트 $f_t$만 통과하면 됩니다. 학습을 통해 $f_t \approx 1$로 유지되면, 기울기는 거의 감쇠 없이 먼 과거까지 흘러갈 수 있습니다. 이것이 LSTM이 장기 의존성을 훨씬 잘 학습하는 구조적 이유입니다.

## 직접 보는 LSTM

아래 시각화에서 **셀 상태(보라)**와 **히든 상태(주황)**가 타임스텝마다 어떻게 바뀌는지 관찰해보세요.

"What's the most resilient parasite" — Inception (2010) 의 각 단어를 1차원 임베딩으로 단순화해 처리합니다.

> **주목 포인트**
> - **셀 상태 c** : 잊기 게이트가 높게 유지(≈0.8)되어 값이 비교적 천천히 변합니다 — 컨베이어 벨트처럼
> - **히든 상태 h** : 매 스텝 셀 상태를 필터링해 새로 만들어지므로 더 빠르게 변합니다
> - **"parasite"** (x = −0.3) : 입력값이 음수라 입력 게이트 i가 낮아지고, 후보값 g도 음수가 됩니다. 셀 상태가 소폭 감소하는 것을 확인하세요
> - 점선 **+** 화살표는 $i_t \cdot g_t$ 가 셀 상태에 더해지는 흐름을 나타냅니다

## PyTorch 코드

### LSTMCell — 타임스텝을 직접 순회하기

```python
import torch
import torch.nn as nn

# 스칼라 입력(1차원) → 은닉 4차원
cell = nn.LSTMCell(input_size=1, hidden_size=4)

tokens = ["What's", "the", "most", "resilient", "parasite"]
x_seq  = torch.tensor([[0.8], [0.1], [0.6], [0.9], [-0.3]])  # (5, 1)

h = torch.zeros(1, 4)   # 초기 히든 상태
c = torch.zeros(1, 4)   # 초기 셀 상태

print(f"{'t':>2}  {'단어':<12}  {'‖c_t‖':>8}  {'‖h_t‖':>8}  {'Δc':>8}  {'Δh':>8}")
print("─" * 55)

for t, (x, word) in enumerate(zip(x_seq, tokens)):
    c_prev, h_prev = c.clone(), h.clone()
    h, c = cell(x.unsqueeze(0), (h, c))   # h, c 동시에 갱신

    dc = (c - c_prev).norm().item()
    dh = (h - h_prev).norm().item()
    print(f" {t}  {word:<12}  {c.norm().item():8.4f}  {h.norm().item():8.4f}  {dc:8.4f}  {dh:8.4f}")
```

실행하면 셀 상태(‖c_t‖)가 히든 상태(‖h_t‖)보다 더 완만하게 변하는 것을 확인할 수 있습니다.

### nn.LSTM — 시퀀스를 한 번에 처리하기

```python
import torch
import torch.nn as nn

# num_layers=2 로 2층 스택 LSTM
lstm = nn.LSTM(
    input_size=10,
    hidden_size=64,
    num_layers=2,
    batch_first=True,   # 입력 shape: (batch, seq, features)
    dropout=0.2,
)

batch, seq_len = 32, 20
x = torch.randn(batch, seq_len, 10)

# output : (batch, seq_len, hidden_size) — 모든 시점의 h_t
# h_n    : (num_layers, batch, hidden_size) — 마지막 히든 상태
# c_n    : (num_layers, batch, hidden_size) — 마지막 셀 상태
output, (h_n, c_n) = lstm(x)

print(f"output shape : {tuple(output.shape)}")   # (32, 20, 64)
print(f"h_n shape    : {tuple(h_n.shape)}")      # (2, 32, 64)
print(f"c_n shape    : {tuple(c_n.shape)}")      # (2, 32, 64)
```

`output`에는 **모든 시점의 히든 상태**가 담겨 있습니다. Many-to-One 분류(감정 분석 등)라면 `output[:, -1, :]` (마지막 시점)이나 `h_n[-1]`을 사용하고, Many-to-Many(번역 등)라면 `output` 전체를 활용합니다.

## LSTM vs RNN 비교

| 항목 | RNN | LSTM |
|------|-----|------|
| 상태 | 히든 상태 $h_t$ 1개 | 셀 상태 $c_t$ + 히든 상태 $h_t$ 2개 |
| 장기 의존성 | 취약 (기울기 소실) | 강함 (셀 상태 고속도로) |
| 파라미터 수 | 적음 | RNN의 약 4배 (게이트 4개) |
| 계산 비용 | 낮음 | 높음 |
| 주요 용도 | 짧은 시퀀스 | 긴 시퀀스, 번역, 언어 모델 |

## 정리

LSTM의 핵심을 한 문장으로 정리하면 이렇습니다.

> LSTM은 셀 상태라는 별도 고속도로로 장기 기억을 보존하고, 세 개의 게이트로 무엇을 기억하고 잊고 출력할지 학습한다.

두 상태의 역할을 다시 한번 짚어봅시다.

- **셀 상태 $c_t$** : 게이트가 특별히 개입하지 않으면 거의 그대로 흘러가는 장기 기억. 기울기도 함께 잘 흘러갑니다.
- **히든 상태 $h_t$** : 현재 시점에 필요한 정보를 셀 상태에서 걸러낸 단기 작업 기억이자 출력 신호.

이 구조 덕분에 LSTM은 수백 타임스텝 이상의 긴 의존 관계도 학습할 수 있으며, 2010년대 NLP의 핵심 아키텍처로 자리잡았습니다. 이후 GRU가 두 상태를 하나로 합쳐 더 간결하게 만들고, Transformer가 순환 구조 자체를 버리며 새 시대를 열게 됩니다.
