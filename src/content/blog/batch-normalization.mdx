# 훈련이 흔들리는 이유

Batch Normalization이 왜 필요했는지 이해하려면, 먼저 신경망 안에서 값들이 어떻게 흘러가는지 보는 것이 좋습니다.

아래 그림을 왼쪽에서 오른쪽으로 따라가 봅시다. 각 박스는 신경망의 한 층이고, 위의 곡선은 그 층에서 나온 값들이 어떤 모양으로 퍼져 있는지를 나타냅니다.

여기서 곡선이 나타내는 값들을 **activation**이라고 부르겠습니다. activation은 가중치가 아닙니다. 가중치는 모델이 학습해서 들고 있는 계산 규칙이고, activation은 그 규칙을 입력에 적용한 뒤 나온 중간 결과값입니다.

예를 들어 사진을 보는 모델이라면, 앞쪽 층은 픽셀을 조합해 "밝기", "테두리", "그림자" 같은 중간 신호를 만듭니다. 이때 어떤 사진에서 밝기 특징이 `5.2`만큼 강하게 나왔다면, 그 `5.2`가 activation입니다.

문제는 학습 중에 앞쪽 층의 가중치가 계속 바뀐다는 점입니다. Layer 1이 조금 바뀌면 Layer 2로 넘어가는 activation의 평균 위치가 달라질 수 있고, Layer 2가 다시 바뀌면 Layer 3이 받는 값의 폭도 달라질 수 있습니다.

그림의 파란 곡선이 점점 오른쪽으로 밀리고, 주황색 곡선이 더 넓게 퍼지는 모습이 바로 이 상황입니다. 뒤쪽 층 입장에서는 어제까지는 "대부분 3 근처에 모이는 값"을 보다가, 오늘은 갑자기 "대부분 8 근처에 모이고 폭도 넓어진 값"을 받는 셈입니다.

여기서 **분포(distribution)** 는 값들이 어느 위치에 얼마나 퍼져 있는지를 뜻합니다. 평균이 어디에 있는지, 값들이 좁게 모여 있는지, 넓게 흩어져 있는지까지 포함한 전체 모양입니다.

이제 BatchNorm의 핵심 질문은 아주 단순합니다.

> 뒤쪽 층이 매번 흔들리는 분포를 받지 않도록, activation의 위치와 폭을 중간에서 한 번 정리해주면 어떨까?

Batch Normalization은 이 질문에서 출발합니다. 그림 오른쪽의 초록색 분포처럼, 흩어지고 밀린 activation을 다시 비교적 안정된 기준으로 맞춰주는 것이 BatchNorm의 기본 역할입니다.

## Batch Normalization이 하는 일

Batch Normalization, 줄여서 BatchNorm은 한 층에서 나온 activation들을 mini-batch 단위로 정규화합니다.

**mini-batch** 는 전체 학습 데이터를 한 번에 쓰지 않고, 몇 개씩 묶어서 학습할 때의 작은 묶음입니다. 예를 들어 batch size가 32라면 한 번의 업데이트에 32개의 샘플을 사용합니다.

한 mini-batch 안에 사진 5장이 있다고 해봅시다. 어떤 층의 한 feature가 각 사진에서 "밝기 특징이 얼마나 강한지"를 계산했고, 그 결과가 아래처럼 나왔습니다.

여기서 **feature**는 모델 내부에서 하나의 특징을 나타내는 값 묶음입니다. CNN이라면 채널 하나, 일반적인 신경망이라면 뉴런 하나의 출력이라고 생각하면 됩니다.

| 샘플 | activation |
| --- | ---: |
| 1 | 5.2 |
| 2 | 4.7 |
| 3 | 6.1 |
| 4 | 2.9 |
| 5 | 3.8 |

BatchNorm이 직접 다루는 것은 가중치가 아니라 이 activation 값들입니다. 먼저 이 mini-batch 안에서 평균과 분산을 구합니다.

$$
\mu_B = \frac{1}{m}\sum_{i=1}^{m} x_i
$$

$$
\sigma_B^2 = \frac{1}{m}\sum_{i=1}^{m}(x_i - \mu_B)^2
$$

여기서 $m$은 mini-batch 안에 들어 있는 샘플 수입니다. $\mu_B$는 batch 평균, $\sigma_B^2$는 batch 분산입니다.

그 다음 각 값을 평균 0, 표준편차 1에 가깝게 바꿉니다.

$$
\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}
$$

$\epsilon$은 아주 작은 숫자입니다. 분산이 0에 가까울 때 나눗셈이 불안정해지는 것을 막기 위해 더합니다. 여기서 **불안정하다**는 말은 컴퓨터가 너무 작은 수로 나누면서 값이 지나치게 커지거나 계산 오류에 가까워지는 상황을 뜻합니다.

4. **gamma와 beta 바꾸기**: gamma는 폭을 바꾸고, beta는 중심 위치를 옮깁니다.

## 그런데 왜 다시 gamma와 beta가 필요할까?

정규화만 하면 모든 feature가 항상 평균 0, 표준편차 1 근처에 머뭅니다. 이것은 학습을 안정시키는 데 좋지만, 모델 입장에서는 너무 강한 제약일 수 있습니다.

어떤 층은 activation의 폭이 조금 더 넓어야 표현이 잘 될 수 있습니다. 어떤 feature는 중심이 0이 아니라 양수 쪽에 있어야 다음 층에서 쓰기 좋을 수도 있습니다.

그래서 BatchNorm은 정규화한 뒤에 다시 학습 가능한 두 파라미터를 붙입니다.

$$
y_i = \gamma \hat{x}_i + \beta
$$

| 기호 | 역할 | 시각적으로 보면 |
| --- | --- | --- |
| $\gamma$ | scale, 폭 조절 | 점구름을 세로로 늘리거나 줄임 |
| $\beta$ | shift, 위치 조절 | 점구름 전체를 위아래로 이동 |

즉 BatchNorm은 “무조건 평균 0, 표준편차 1로 고정하자”가 아닙니다. 먼저 계산하기 편한 기준점으로 정리한 뒤, 모델이 필요하면 다시 적절한 분포를 학습하게 합니다.

## 학습을 왜 도와줄까?

BatchNorm이 학습을 돕는 이유를 하나로만 설명하기는 어렵습니다. 원 논문은 내부 공변량 변화, 즉 앞쪽 층의 변화 때문에 뒤쪽 층 입력 분포가 바뀌는 문제를 줄인다고 설명했습니다.

**공변량(covariate)** 은 모델에 들어가는 입력 변수를 뜻합니다. **내부 공변량 변화(internal covariate shift)** 는 모델 내부 층들이 받는 입력 분포가 학습 중 계속 바뀌는 현상을 가리킵니다.

다만 현대적인 해석에서는 BatchNorm의 효과를 조금 더 넓게 봅니다.

- activation의 평균과 스케일이 극단적으로 튀는 것을 줄입니다.
- loss landscape를 더 완만하게 만들어 큰 learning rate를 쓰기 쉬워집니다.
- mini-batch 통계가 약간의 noise를 만들기 때문에 정규화 효과도 생깁니다.

여기서 **loss landscape** 는 파라미터 위치마다 loss가 얼마나 큰지 그린 지형도라고 생각하면 됩니다. 지형이 너무 뾰족하고 울퉁불퉁하면 작은 이동에도 loss가 크게 변해서 학습이 까다롭습니다.

## 훈련 때와 추론 때는 다르다

BatchNorm에서 가장 많이 헷갈리는 부분은 train mode와 eval mode입니다.

훈련 중에는 현재 mini-batch의 평균과 분산을 사용합니다. 하지만 추론할 때, 즉 이미 학습된 모델로 예측만 할 때는 보통 batch size가 1일 수도 있습니다. 이때 현재 batch 통계를 그대로 쓰면 예측이 입력 묶음에 따라 달라질 수 있습니다.

그래서 BatchNorm은 훈련 중에 **running mean**과 **running variance**를 함께 저장합니다.

| 모드 | 사용하는 평균/분산 | 이유 |
| --- | --- | --- |
| train | 현재 mini-batch 통계 | 학습 중 activation을 즉시 안정화 |
| eval | 훈련 중 누적한 running 통계 | 예측 결과가 batch 구성에 덜 흔들리게 함 |

**running**은 “계속 업데이트하며 누적한다”는 뜻입니다. PyTorch의 `BatchNorm`도 학습 중 batch 통계를 보면서 running mean과 running variance를 조금씩 갱신하고, `model.eval()` 상태에서는 이 누적 통계를 사용합니다.

## CNN에서는 어디에 놓을까?

CNN에서 BatchNorm은 보통 convolution 다음에 놓습니다.

$$
\mathrm{Conv} \rightarrow \mathrm{BatchNorm} \rightarrow \mathrm{ReLU}
$$

Convolution은 이미지에서 작은 필터를 움직이며 feature map을 만드는 연산입니다. BatchNorm은 이 feature map의 채널별 activation 분포를 안정시킵니다. 그 다음 ReLU 같은 activation function이 비선형성을 추가합니다.

다만 모든 모델에서 BatchNorm이 정답은 아닙니다. Transformer나 작은 batch로 학습하는 경우에는 Layer Normalization, Group Normalization 같은 다른 정규화가 더 자주 쓰입니다. 특히 batch size가 너무 작으면 batch 평균과 분산 추정 자체가 불안정해질 수 있습니다.

## 조심해야 할 점

BatchNorm은 강력하지만 몇 가지 주의점이 있습니다.

- batch size가 너무 작으면 통계 추정이 흔들립니다.
- train/eval mode를 잘못 설정하면 validation 또는 inference 결과가 이상해질 수 있습니다.
- recurrent model이나 Transformer에서는 BatchNorm보다 LayerNorm이 더 자연스러운 경우가 많습니다.
- dropout과 함께 쓸 때는 보통 `Conv/Linear → BatchNorm → Activation → Dropout` 순서를 먼저 고려합니다.

여기서 **inference** 는 학습이 끝난 모델을 실제 입력에 적용해 예측값을 얻는 단계입니다. validation은 학습 중 모델이 잘 일반화되는지 확인하기 위해 따로 빼둔 데이터에서 평가하는 단계입니다.

## 정리

Batch Normalization의 핵심은 activation을 “매번 비슷한 기준 좌표계”로 보내는 것입니다.

1. mini-batch에서 평균과 분산을 구합니다.
2. 평균을 빼서 중심을 0 근처로 옮깁니다.
3. 표준편차로 나눠 폭을 1 근처로 맞춥니다.
4. gamma와 beta로 모델이 필요한 폭과 위치를 다시 학습합니다.
5. 훈련 때는 batch 통계를, 추론 때는 running 통계를 사용합니다.

시각적으로 기억하면 더 간단합니다. BatchNorm은 흩어진 activation 점구름을 잠깐 가지런히 세운 다음, 모델이 원하는 만큼 다시 펼쳐주는 층입니다.
