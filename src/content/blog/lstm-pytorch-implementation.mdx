## PyTorch 코드

이번에는 더 현실적인 형태로 LSTM을 직접 구현해보겠습니다.  
`PyTorch`는 딥러닝 모델을 코드로 만드는 파이썬 라이브러리이고, `forward`는 입력을 넣었을 때 출력이 계산되는 한 번의 진행입니다.

아래 코드는 감정 분석 모델을 **학습**하는 예제가 아니라, 문장 하나가 LSTM 안에서 어떻게 흘러가는지 관찰하기 위한 예제입니다.

### LSTMCellManual — 한 타임스텝만 처리하는 LSTM 셀

이번 버전은 수식을 더 곧바로 읽을 수 있게 정리한 코드입니다.  
먼저 `LSTMCell` 하나를 직접 만들고, 그다음 이 셀들을 묶은 `LSTMManual` 레이어를 만든 뒤, `"Pain past is pleasure ."` 문장을 넣어 봅니다.

여기서 `타임스텝(timestep)`은 시퀀스를 왼쪽에서 오른쪽으로 한 칸씩 읽는 순서를 뜻합니다.

```python
import torch
import torch.nn as nn
class LSTMCellManual(nn.Module):
    """
    LSTM Cell 수식을 그대로 구현.
    입력: x(t), h(t-1), c(t-1)
    출력: h(t), c(t)
    """
    def __init__(self, input_size, hidden_size):
        super().__init__()
        self.hidden_size = hidden_size

        # 게이트 4개(f, i, g, o)를 한 번에 계산하기 위해
        # 가중치를 4*hidden_size 크기로 묶음 (PyTorch 관행)
        self.W_ih = nn.Linear(input_size,  4 * hidden_size, bias=True)   # x → gates
        self.W_hh = nn.Linear(hidden_size, 4 * hidden_size, bias=False)  # h → gates

    def forward(self, x, h_prev, c_prev):
        # ── 게이트 계산 (4개 한번에) ──────────────────────────
        gates = self.W_ih(x) + self.W_hh(h_prev)   # (batch, 4*hidden)

        # chunk로 forget / input / candidate / output 분리
        f, i, g, o = gates.chunk(4, dim=-1)

        # ── 각 게이트 활성화 ──────────────────────────────────
        f = torch.sigmoid(f)        # forget gate    : 과거를 얼마나 잊을지
        i = torch.sigmoid(i)        # input gate     : 새 정보를 얼마나 받을지
        g = torch.tanh(g)           # candidate      : 실제 새 정보 내용
        o = torch.sigmoid(o)        # output gate    : 출력 범위 결정

        # ── Cell state 업데이트 ───────────────────────────────
        c_next = f * c_prev + i * g          # 과거 유지 + 새 정보 추가

        # ── Hidden state 출력 ─────────────────────────────────
        h_next = o * torch.tanh(c_next)      # cell state를 압축 후 출력

        return h_next, c_next
```

### LSTMManual — 시퀀스 전체를 처리하는 레이어

이제 방금 만든 `LSTMCellManual`을 여러 시점에 반복해서 적용하는 `LSTMManual`을 만듭니다.  
여기서 `레이어(layer)`는 같은 종류의 계산 블록을 한 층으로 묶은 것이고, `num_layers=1`이면 한 겹짜리 LSTM이라는 뜻입니다.

```python
class LSTMManual(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers=1, batch_first=True):
        super().__init__()
        self.hidden_size = hidden_size
        self.num_layers  = num_layers
        self.batch_first = batch_first

        self.cells = nn.ModuleList([
            LSTMCellManual(
                input_size if i == 0 else hidden_size,
                hidden_size
            )
            for i in range(num_layers)
        ])

    def forward(self, x, hidden=None):
        # x shape: (batch, seq, input) if batch_first
        if self.batch_first:
            x = x.transpose(0, 1)    # -> (seq, batch, input)

        seq_len, batch_size, _ = x.shape

        if hidden is None:
            h = [torch.zeros(batch_size, self.hidden_size, device=x.device)
                 for _ in range(self.num_layers)]
            c = [torch.zeros(batch_size, self.hidden_size, device=x.device)
                 for _ in range(self.num_layers)]
        else:
            h = list(hidden[0].unbind(0))
            c = list(hidden[1].unbind(0))

        outputs = []
        for t in range(seq_len):
            layer_input = x[t]

            for layer_idx, cell in enumerate(self.cells):
                h[layer_idx], c[layer_idx] = cell(
                    layer_input,
                    h[layer_idx],
                    c[layer_idx]
                )
                layer_input = h[layer_idx]

            outputs.append(h[-1])

        output = torch.stack(outputs, dim=0)   # (seq, batch, hidden)
        if self.batch_first:
            output = output.transpose(0, 1)    # (batch, seq, hidden)

        hn = torch.stack(h, dim=0)
        cn = torch.stack(c, dim=0)

        return output, (hn, cn)
```

### 문장 하나를 임베딩해서 넣어보기

이번에는 `"Pain past is pleasure ."` 문장을 `embedding`으로 바꿔서, `LSTMManual`에 넣어 보겠습니다.  
여기서 `embedding`은 단어를 숫자 벡터로 바꾼 표현입니다.

```python
torch.manual_seed(42)

# 문장 하나 준비
sentence = "Pain past is pleasure ."
tokens = sentence.split()

# 단어 사전 만들기
vocab = sorted(set(tokens))
word2idx = {word: idx for idx, word in enumerate(vocab)}
input_ids = torch.tensor([word2idx[token] for token in tokens], dtype=torch.long)

print("tokens:", tokens)
print("word2idx:", word2idx)
print("input_ids:", input_ids)

# embedding과 LSTM 준비
embedding_dim = 5
hidden_size = 3

embedding = nn.Embedding(
    num_embeddings=len(vocab),
    embedding_dim=embedding_dim
)

lstm = LSTMManual(
    input_size=embedding_dim,
    hidden_size=hidden_size,
    num_layers=1,
    batch_first=True
)

# token id를 embedding vector로 바꾸기
embeddings = embedding(input_ids)   # (seq_len, embedding_dim)

# batch 차원을 붙여서 (1, seq_len, embedding_dim)으로 만듭니다.
x = embeddings.unsqueeze(0)

# LSTMManual에 전체 시퀀스를 넣습니다.
output, (hn, cn) = lstm(x)

print("embeddings shape:", embeddings.shape)   # (5, 5)
print("x shape:", x.shape)                     # (1, 5, 5)
print("output shape:", output.shape)           # (1, 5, 3)
print("hn shape:", hn.shape)                   # (1, 1, 3)
print("cn shape:", cn.shape)                   # (1, 1, 3)
```

위 코드를 실행하면 `"Pain" → "past" → "is" → "pleasure" → "."` 순서대로 단어를 읽으면서, hidden state와 cell state가 차례로 갱신됩니다.  
`output`에는 모든 시점의 hidden state가 쌓이고, `hn`과 `cn`에는 마지막 시점의 hidden state와 cell state가 들어갑니다.

참고로, 아래 시각화는 구조를 보여주기 위해 **한 타임스텝의 LSTM 셀 내부**를 확대해 설명합니다.  
즉 본문 코드에서는 `LSTMManual`이 전체 시퀀스를 처리하고, 시각화는 그 안에서 반복되는 `LSTMCellManual` 한 번의 계산을 보여준다고 생각하면 됩니다.

실무에서는 같은 구조를 직접 구현하기보다 PyTorch가 제공하는 `nn.LSTM`이나 `nn.LSTMCell`을 더 자주 사용합니다. 다만 구조를 이해할 때는 위처럼 직접 셀을 한 번 써보는 것이 가장 빠릅니다.

## 직접 보는 LSTM

아래 시각화는 위 코드처럼 **단일 LSTM 셀에 토큰을 하나씩 넣는 흐름**을 단순화해서 보여주는 보조 시각화입니다.

> **주목 포인트**
> - **▶ 재생**을 누르면 `x_t → 4개 gate → c_t → h_t` 순서로 단계가 진행됩니다.
> - **벡터 표**에서 현재 단계 행이 강조됩니다. `dim`은 벡터의 각 칸 번호, 즉 벡터 안 각 숫자의 위치입니다.
> - **셀 상태 `c_t`** 는 장기 기억 통로라서, 과거 기억을 남기면서 천천히 누적됩니다.
> - **히든 상태 `h_t`** 는 현재 시점에 필요한 정보만 밖으로 드러낸 작업 메모리입니다.
> - 이 예제는 학습 전 랜덤 가중치 상태이므로, 숫자 자체에 문장 의미를 직접 부여하면 안 됩니다.
